Order Processing



Customers can add their orders using online upload tool or using Customer API system. When a new order saved into the database, the system automatically try to process the order.



The system checks the following attributes for each products in the order file or API call.

  • if the product is existing in warehouse and stock level is enough for ordered quantity
  • if the product has a price group that matching with customer's price group
  • if the product has a stock group that matching with customer's stock group




If the system can validate all these steps in order, start to process the order. First updates the stock levels in Warehouse and update the order status. If it fails, you may always process the order manually from backoffice. The system creates the notes in order file why the ordered product failed in the automated process. If you see that price or stock group is wrong, you may update it. If the stock level is not enough, you may manage the stocks in warehouse module to let the customer order the product.
